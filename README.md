# English Random Card — 작업 매뉴얼

영어 **상황(situation)** 카드를 매일 랜덤으로 연습하는 단일 페이지 앱을 만듭니다. 이 문서는 구조·데이터 형식·일일 노출 규칙을 정의합니다.

## 목표

- **학습 흐름**: 카드 **앞면**에는 상황만 보이고, 사용자가 말한 뒤 **클릭**하면 **영어 모범 답변**을 확인합니다.
- **일일 분량**: 하루 **3장**씩만 진행합니다.
- **데이터**: 상황과 답은 **JSON**에 두고, 내용은 직접 입력·유지합니다.

## 파일 구조


| 파일                              | 역할                                    |
| ------------------------------- | ------------------------------------- |
| `index.html`                    | UI·동작(카드 표시, 클릭 시 뒤집기/답 표시, 일일 3장 로직) |
| `data/*.json` (또는 프로젝트에서 정한 경로) | 영어 학습용 카드 데이터 (아래 스키마)                |


경로는 구현 시 하나로 고정하고, 이 README와 동일하게 유지합니다.

## JSON 데이터 스키마

각 항목은 최소한 다음 필드를 갖습니다.

```json
{
  "id": "unique-string",
  "situation": "English situation shown on the front of the card.",
  "answer": "English model response for that situation."
}
```

- `**situation**`: 카드 앞면에만 노출되는 **상황** 문장.
- `**answer`**: 클릭 후 확인하는 **영어 대답**(모범 답안).
- `**id`**: 일일 추첨·로컬 저장(오늘 본 카드 등)에 쓰기 위한 고유 식별자. 없으면 배열 인덱스로 대체할 수 있으나, 데이터 편집 시 충돌을 피하려 `id` 권장.

배열 예시:

```json
[
```


  { "id": "1", "situation": "...", "answer": "..." },
  { "id": "2", "situation": "...", "answer": "..." }
]

**정렬 규칙**: 배열에서 **뒤쪽일수록 최신**으로 간주합니다. (새 카드는 배열 끝에 추가.)

## 화면·UX (index.html)

1. 그날 선택된 **3장** 중 현재 카드를 표시합니다.
2. **앞면**: `situation`만 표시합니다.
3. 사용자가 스스로 말한 뒤, **클릭**(또는 동일 의미의 버튼)하면 **답**을 보여줍니다 (`answer`).
4. 다음 카드로 넘기거나, 3장을 모두 마치면 오늘 세션 완료 처리합니다.

(카드 애니메이션·진행 표시 등은 `index.html` 구현 범위에서 결정합니다.)

## 하루 3장 선정 규칙

하루에 보여줄 **3개**는 아래 **세 구간**에서 뽑으며, 구간별 **가중치**는 다음과 같습니다.


| 구간          | 범위 (데이터 기준)                                                   | 가중치    |
| ----------- | ------------------------------------------------------------- | ------ |
| **최신**      | 전체 항목 중 **가장 최근에 추가된 3개** (배열의 마지막 3개)                        | **10** |
| **이전 회차**   | 그 바로 앞 구간 (최신 3개 **직전**에 해당하는 한 덩어리; 예: 그 앞 3개 또는 구현에서 정한 묶음) | **6**  |
| **그 이전 회차** | 더 과거에 해당하는 나머지(또는 그 이전 묶음)                                    | **3**  |


- 하루 **3장**을 고를 때, 위 세 구간에 대해 **10 : 6 : 3** 비율이 반영되도록 샘플링합니다. (예: 가중 랜덤으로 3슬롯을 채우거나, 동일 규칙을 만족하는 방식으로 구현.)
- 항목이 3개 미만이면 가능한 만큼만 표시하고, 구간이 비어 있으면 다른 구간에서만 채웁니다.
- **같은 날 같은 카드 중복** 여부(허용/금지)는 구현 시 정하고, 여기에 한 줄로 적어 두면 이후 수정이 쉽습니다.

## 작업 체크리스트

- `index.html` 생성 및 기본 레이아웃
- JSON 로드 → 파싱 → 유효성 검사 (`situation`, `answer` 존재)
- “오늘 날짜” 기준 일일 3장 선정 (가중치 10 / 6 / 3, 구간 정의와 동일하게)
- 앞면 상황 → 클릭 시 답 표시 → 3장 완료 플로우
- (선택) 오늘 완료 여부를 `localStorage` 등에 저장해 새로고침 후에도 유지

## 용어 정리

- **상황**: 사용자가 말로 대응해야 하는 영어 프롬프트(앞면).
- **영어 대답**: 그 상황에 대한 모범 영어 응답(클릭 후 확인).
- **회차**: 데이터가 쌓이는 순서를 말할 때, “최신 3개 / 그 이전 묶음 / 그보다 이전”으로 나눈 구간을 의미합니다.
