# English Random Card — 작업 매뉴얼

영어 **상황(situation)** 카드를 매일 랜덤으로 연습하는 단일 페이지 앱을 만듭니다. 이 문서는 구조·데이터 형식·일일 노출 규칙을 정의합니다.

## 목표

- **학습 흐름**: 카드 **앞면**에는 상황만 보이고, 사용자가 말한 뒤 **클릭**하면 **영어 모범 답변**을 확인합니다.
- **일일 분량**: 하루 **3장**씩만 진행합니다.
- **데이터**: 상황과 답은 **JSON**에 두고, **`studyDate`(달력 날짜)**로 “언제 넣은 묶음인지”를 구분합니다.

## 파트 구성

- `data/cards.json`: **일상 생활**
- `data/truth.json`: **진리발표**

## 파일 구조

| 파일 | 역할 |
|------|------|
| `index.html` | UI·동작(카드 표시, 클릭 시 답 표시, 일일 3장 로직) |

경로는 구현 시 하나로 고정하고, 이 README와 동일하게 유지합니다.

## JSON 데이터 스키마

각 항목은 최소한 다음 필드를 갖습니다.

```json
{
  "id": "unique-string",
  "round": 1,
  "studyDate": "2026-04-19",
  "situation": "상황 설명 (예: 한국어)",
  "answer": "English model response for that situation."
}
```

- **`studyDate`**: **`YYYY-MM-DD`** 문자열. 기기 **달력 “오늘”**과 비교해 **6 · 4 · 2** 순차 게이트와 가중 추첨 구간을 나눕니다. 같은 날에 추가한 카드 3장은 보통 **같은 `studyDate`**를 갖게 하면 됩니다.
- **`round` (회차, 선택적 표시용)**: UI에서 묶음을 보여 줄 때 쓸 수 있는 정수. **`round`가 클수록 최신**으로 두되, **추첨·게이트의 기준은 `studyDate`가 우선**입니다.
- **`situation`**: 카드 앞면에만 노출되는 **상황** 문장.
- **`answer`**: 클릭 후 확인하는 **영어 대답**(모범 답안).
- **`id`**: 고유 문자열. **`id` 순서는 연습 순서와 무관**합니다.

### 진리발표 데이터 (`data/truth.json`)

진리발표는 날짜 게이트가 아니라 **챕터 선택 학습**을 기준으로 합니다.

```json
{
  "chapter": 7,
  "title": "Passover, the Way to Eternal Life",
  "cards": [
    {
      "id": 1,
      "situation": "새언약의 유월절은",
      "answer": "The Passover of the new covenant"
    }
  ]
}
```

## 화면·UX (index.html)

1. 그날 선택된 **3장** 중 현재 카드를 표시합니다.
2. **앞면**: `situation`만 표시합니다.
3. 사용자가 스스로 말한 뒤, **클릭**하면 **답**을 보여줍니다 (`answer`).
4. **「다음 카드」/「완료」**를 누르면 해당 카드의 **완료 횟수**가 1 증가합니다(진도 저장).

**표시 순서**: 그날 뽑힌 3장의 **1→2→3번째 순서**는 셔플로 무작위입니다.

## 날짜 기준 6 · 4 · 2 (순차 게이트)

기기 로컬 날짜를 **오늘 `T`**, **`T - 1`일**, **`T - 2`일**로 둡니다.

1. **`studyDate === T`** 인 카드가 하나라도 있으면: 그 카드들만 일일 출제 대상에 쓸 수 있고, **각 카드마다** “답 확인 → 다음/완료”를 **6회** 채울 때까지 이 단계에 머뭅니다.
2. 위가 모두 끝나면 **`studyDate === T - 1`** 인 카드가 있을 때: 그 카드들만 쓰며, **각 카드마다 4회**를 채웁니다.
3. 그다음 **`studyDate === T - 2`** 인 카드가 있을 때: **각 카드마다 2회**를 채웁니다.

해당 날짜에 카드가 **없으면** 그 단계는 건너뜁니다(예: 오늘 날짜 카드가 없으면 바로 “어제” 단계부터).

## 하루 3장 선정 규칙 (게이트 통과 후)

위 **순차 게이트를 모두 통과한 뒤에만**, `studyDate`를 오늘 / 어제 / 그보다 이전으로 나누고 **가중치 6 : 4 : 2**로 세 구간에서 뽑습니다.

| 구간 | 조건 (`studyDate` 기준, 오늘을 `T`) | 가중치 |
|------|-------------------------------------|--------|
| **오늘** | `studyDate === T` | **6** |
| **어제** | `studyDate === T - 1` | **4** |
| **그 이전** | `studyDate < T - 1` | **2** |

- 하루 **3장**은 위 비율로 샘플링하고, 구간이 비면 건너뜁니다.
- **같은 날 같은 카드 중복**은 금지합니다.
- **진도**(카드별 완료 횟수)는 `localStorage`에 저장됩니다.
