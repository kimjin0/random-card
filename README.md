# English Random Card — 작업 매뉴얼

영어 **상황(situation)** 카드를 매일 랜덤으로 연습하는 단일 페이지 앱을 만듭니다. 이 문서는 구조·데이터 형식·일일 노출 규칙을 정의합니다.

## 목표

- **학습 흐름**: 카드 **앞면**에는 상황만 보이고, 사용자가 말한 뒤 **클릭**하면 **영어 모범 답변**을 확인합니다.
- **일일 분량**: 하루 **3장**씩만 진행합니다.
- **데이터**: 상황과 답은 **JSON**에 두고, 내용은 직접 입력·유지합니다. (상황은 한국어·영어 등 자유, 답은 보통 영어.)

## 파일 구조

| 파일 | 역할 |
|------|------|
| `index.html` | UI·동작(카드 표시, 클릭 시 답 표시, 일일 3장 로직) |
| `data/*.json` | 영어 학습용 카드 데이터 (아래 스키마) |

경로는 구현 시 하나로 고정하고, 이 README와 동일하게 유지합니다.

## JSON 데이터 스키마

각 항목은 최소한 다음 필드를 갖습니다.

```json
{
  "id": "unique-string",
  "round": 1,
  "situation": "상황 설명 (예: 한국어)",
  "answer": "English model response for that situation."
}
```

**시간 순서(고정 규칙)**: `round` 숫자가 **클수록 최신 회차**이고, **작을수록 과거 회차**입니다. “최신 회차”는 항상 데이터 안의 **`round` 최댓값 `R`**입니다.

- **`round` (회차)**: 하루 복습 추첨에서 **10 / 6 / 3** 비중을 나눌 때의 기준입니다. **가장 큰 `round` = `R` = 가장 최신 회차**입니다.
- **`situation`**: 카드 앞면에만 노출되는 **상황** 문장.
- **`answer`**: 클릭 후 확인하는 **영어 대답**(모범 답안).
- **`id`**: 일일 추첨·`localStorage`에 쓰이는 **고유 문자열**입니다. **`id`의 접두어·번호(예: `r1-1`)는 “연습 순서”를 뜻하지 않습니다.** JSON에 적힌 배열 순서와도 무관하게, 오늘의 1·2·3번째 카드 순서는 아래 **표시 순서** 규칙대로 무작위로 정해집니다.

배열 예시:

```json
[
  { "id": "a", "round": 1, "situation": "...", "answer": "..." },
  { "id": "b", "round": 2, "situation": "...", "answer": "..." }
]
```

**회차 = 하루 분량 3장(권장)**: 한 `round` 값에는 카드 **3장**을 두는 방식을 기본으로 합니다. 새 날마다 `round`를 1씩 올리며 3장씩 쌓습니다.

## 화면·UX (index.html)

1. 그날 선택된 **3장** 중 현재 카드를 표시합니다.
2. **앞면**: `situation`만 표시합니다.
3. 사용자가 스스로 말한 뒤, **클릭**(또는 동일 의미의 버튼)하면 **답**을 보여줍니다 (`answer`).
4. 다음 카드로 넘기거나, 3장을 모두 마치면 오늘 세션 완료 처리합니다.

**표시 순서(고정 규칙)**: 가중치로 **오늘의 3장 “멤버”만** 고른 뒤, **1번째·2번째·3번째로 넘길 순서**는 JSON 배열 순서·`id` 사전순·`r1-1` 같은 네이밍과 **무관하게 무작위(셔플)**로 섞습니다. **같은 `round` 안의 3장만 나오는 경우에도** 그날 순서는 매번 랜덤입니다.

## 하루 3장 선정 규칙

매일 화면에 올리는 **3장**은 **`round`(회차)**로 세 구간을 나눈 뒤, 아래 **가중치(비율)**로 “뽑힐 구간”을 정하고 카드를 고릅니다.

1. 데이터에 등장하는 `round` 값 중 **최댓값**을 `R`이라 합니다. (**숫자가 클수록 최신**이므로, 최댓값이 곧 가장 최신 회차입니다.)
2. 각 카드는 아래 중 하나에만 속합니다.

| 구간 | 조건 (`round` 기준) | 가중치 |
|------|---------------------|--------|
| **최신 회차** | `round === R` | **10** |
| **이전 회차** | `round === R - 1` | **6** |
| **그 이전 회차** | `round <= R - 2` | **3** |

- 하루 **3장**을 고를 때, 위 세 구간에 **10 : 6 : 3** 비율이 반영되도록 샘플링합니다.
- 구간에 카드가 없거나 전체가 3장 미만이면, 비어 있는 구간은 건너뛰고 가능한 만큼만 채웁니다.
- **같은 날 같은 카드 중복**: 구현에서는 **금지**합니다.
- 멤버 3장이 정해진 뒤 **표시 순서**는 위 “화면·UX”의 셔플 규칙을 따릅니다.

## 용어 정리

- **상황**: 사용자가 말로 대응해야 하는 프롬프트(앞면).
- **영어 대답**: 그 상황에 대한 모범 영어 응답(클릭 후 확인).
- **회차 (`round`)**: **`round`가 클수록 최신**. `R` = 최댓값, `R-1` = 이전 회차, `round <= R-2` = 그 이전 묶음. **10 / 6 / 3**은 이 세 묶음에 주는 추첨 가중치입니다.
